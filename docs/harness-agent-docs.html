What is HarnessAgent?

Think about what actually happens when you run an AI agent in production. The LLM call needs to work. It needs to not cost $500 a day. It needs to not loop forever when the API is slow. It needs to remember context across sessions. It needs to not crash when one provider goes down.

HarnessAgent handles all of that. You write the task. It handles the rest.

What you seeWhat happens under the hood
AI answers your questionPicks the healthiest LLM, checks the budget, falls back if the provider fails
AI runs a SQL queryValidates input schema, checks safety rules, executes, logs the result
AI remembers past contextShort-term in Redis, long-term in a vector DB, structured in a knowledge graph
AI finds relevant info fastGraph RAG: entity extraction + BFS traversal — 83% fewer tokens than naive vector search
AI gets better after failuresHermes loop: samples errors, proposes a prompt fix, evaluates it, applies if score ≥ 70%
One provider goes downCircuit breaker opens after 5 failures, auto-recovers after 60 seconds

Quick Start

Option A — Cloud LLM (Anthropic or OpenAI)

bash
# 1. Clone and install
git clone https://github.com/thepradip/HarnessAgent.git
cd HarnessAgent
poetry install

# 2. Configure
cp .env.example .env
# Edit .env: set ANTHROPIC_API_KEY or OPENAI_API_KEY

# 3. Start infrastructure
docker compose up -d

# 4. Start API + worker
make api      # terminal 1 -- FastAPI on port 8000
make worker   # terminal 2 -- RQ agent worker

# 5. Run your first agent
curl -X POST http://localhost:8000/runs \
  -H "Content-Type: application/json" \
  -d '{"agent_type": "sql", "task": "How many users signed up this week?"}'

# Stream steps in real time
curl http://localhost:8000/runs/{run_id}/steps

Option B — Local LLM (no API key)

bash
mkdir -p models
# Copy your .gguf file there (e.g. Mistral-7B-Q4)
docker compose --profile local-cpu up -d llamacpp
echo "LLAMACPP_BASE_URL=http://localhost:8080" >> .env
make api && make worker

Option C — Existing LangGraph / AutoGen / CrewAI Agent

python
from harness.adapters.langgraph import LangGraphAdapter
from harness.orchestrator.runner import AgentRunner

adapted = LangGraphAdapter(your_graph)

runner = AgentRunner.from_env()
result = await runner.run_adapted(adapted, task="Analyze Q3 sales data")
✅All three options produce the same result: full MLflow traces, Prometheus metrics, cost tracking, safety guardrails, and circuit breaking — with zero extra code on your part.

Creating Agents

All agents inherit from BaseAgent. Override three things: your system prompt, how you frame the initial task, and optional post-processing. Everything else — the ReAct loop, memory, tool calls, budget checks, checkpointing, observability — is inherited automatically.

1

Subclass BaseAgent

Override system_prompt, build_initial_messages, and optionally post_process.

2

Register in AgentFactory

Add one line mapping your AGENT_TYPE string to the factory lambda.

3

Call POST /runs with your agent_type

Your agent runs with full HarnessAgent capabilities — no extra setup.

pythonsrc/harness/agents/my_agent.py
from __future__ import annotations
from dataclasses import dataclass
from harness.agents.base import BaseAgent
from harness.core.context import AgentContext, AgentResult


@dataclass
class MyAgentConfig:
    max_results: int = 10
    domain: str = "general"
    strict_mode: bool = False


class MyAgent(BaseAgent):
    """Domain-specialized agent. Inherits ReAct loop, memory,
    budget enforcement, safety, tracing, and checkpointing."""

    AGENT_TYPE = "my_agent"  # register this in AgentFactory

    _SYSTEM = """\
You are a specialized {domain} assistant.
Available tools: {tool_list}

Guidelines:
1. Think step-by-step before calling a tool.
2. Always verify results before presenting them.
3. If unsure, ask for clarification -- never guess.
"""

    def __init__(self, config: MyAgentConfig, **kwargs) -> None:
        super().__init__(**kwargs)
        self.config = config

    @property
    def system_prompt(self) -> str:
        return self._SYSTEM.format(
            domain=self.config.domain,
            tool_list=", ".join(self.tool_registry.list_tool_names()),
        )

    def build_initial_messages(
        self, task: str, context: AgentContext
    ) -> list[dict]:
        return [{"role": "user", "content": f"Task: {task}\nMax results: {self.config.max_results}"}]

    async def post_process(self, result: AgentResult) -> AgentResult:
        return result  # optional: transform the final result
pythonsrc/harness/orchestrator/factory.py — add one line
from harness.agents.my_agent import MyAgent, MyAgentConfig

AGENT_REGISTRY = {
    "sql":      lambda deps: SQLAgent(**deps),
    "code":     lambda deps: CodeAgent(**deps),
    "my_agent": lambda deps: MyAgent(config=MyAgentConfig(), **deps),  # add
}
✅Your agent now has: LLM routing with fallback, 3-tier memory, budget enforcement, circuit breaking, safety pipeline, MLflow traces, and Prometheus metrics — zero extra code.

Use Cases

SQL Data Agent

Ask business questions in plain English. The agent reads your schema into a knowledge graph, generates safe SELECT-only queries, validates them, executes, and returns results with PII redacted.

Code Assistant

Give it a ticket or spec. It reads your workspace, writes code, lints it, runs it in a Docker sandbox, and iterates on errors until tests pass. Checkpointing means it survives restarts.

Research Agent

Feed it documents or URLs. It ingests them into the vector store and knowledge graph, then answers multi-hop questions with citations. Graph RAG reduces token usage 83%.

Multi-Agent Pipeline

Chain specialists through the Planner: researcher → coder → reviewer. The Scheduler maximizes parallelism across independent subtasks. All agents share the same memory pool.

Drop-In for Existing Frameworks

Already using LangGraph, AutoGen, or CrewAI? Drop your graph or crew into an adapter. You immediately get traces, cost tracking, circuit breaking, and safety without changing a line of agent logic.

API Reference

The FastAPI server runs on port 8000. All endpoints accept and return JSON. Authentication uses Authorization: Bearer <jwt>. Interactive Swagger docs at http://localhost:8000/docs.

Runs

POST
/runs
Create and queue a new agent run. Returns run_id immediately; execution is async via RQ.
GET
/runs/{run_id}
Get run status, final result, token usage, and cost.
GET
/runs/{run_id}/steps
Stream execution steps as SSE. Subscribe to watch the agent think in real time.
GET
/runs
List recent runs for the authenticated tenant (paginated, newest first).
DEL
/runs/{run_id}
Cancel a queued or running agent. Triggers checkpoint save before stopping.

Memory

POST
/memory/remember
Store a fact or document into long-term memory (vector + graph). PII is auto-redacted.
POST
/memory/recall
Retrieve context for a query using Graph RAG (semantic + graph multi-hop traversal).

Self-Improvement

GET
/improvement/patches
List prompt patches generated by Hermes awaiting review.
POST
/improvement/patches/{id}/apply
Manually apply an approved patch to the prompt store.
GET
/health
Liveness + readiness probes. Returns Redis ping, LLM provider statuses, circuit breaker states.

Learn per-task-type patterns to auto-select the cheapest sufficient model rather than always routing to the most capable one.

↓ Eliminates over-provisioned Opus/GPT-5 usage
Memory

Differential Re-indexing

Re-embed only modified chunks on ingestion. Track document fingerprints to detect changes.

↑ Faster incremental ingestion at scale
Parallelism

Begin tool execution while the LLM is still generating tokens. Reduce idle wait in the ReAct loop.

↓ Lower end-to-end agent step latency
Multi-Agent

Shared Tool Execution Pool

Deduplicate identical tool calls across concurrent agents running in the same session.

↓ Fewer redundant DB and API round-trips
Hermes

↑ Better ROI from self-improvement cycles
